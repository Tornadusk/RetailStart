Data Lake (carpetas)

  • /data_sources/: orígenes dispersos (aquí pones los CSV/JSON/XML/TXT para probar)
  • /data_lake/raw/: landing tras run_elt_ingest (archivos con sufijo de fecha)
  • /data_lake/processed/: salida de run_etl (CSVs con timestamp)
  • /data_lake/processed/evidence/: PNG de analyze_dw

Ver evidencia (CSVs + gráficos): /evidence/

Ver analytics (consultas/tablas desde el DW): /analytics/

Ver flujo completo (Origen → Consumo): /flow/

Comandos Docker (orden recomendado)

Desde la raíz del repo (junto a docker-compose.yml), con docker compose up corriendo. El flujo parte de data_sources/ y regenera raw, processed y el DW.

  1. docker compose exec backend python manage.py run_elt_ingest
  2. docker compose exec backend python manage.py run_etl
  3. docker compose exec backend python manage.py load_dw
  4. docker compose exec backend python manage.py analyze_dw

En Windows las mismas rutas bajo el proyecto: …\data_sources\, …\data_lake\raw\, …\data_lake\processed\, …\data_lake\processed\evidence\.
