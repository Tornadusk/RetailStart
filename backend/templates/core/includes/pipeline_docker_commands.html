{% comment %} Árbol de datos (rúbrica — cargas incrementales Día 1 / Día 2): data_sources/dia_N -> run_elt_ingest -> raw -> run_etl --append-master -> processed/maestro -> load_dw --incremental -> Postgres -> audit_pipeline (evidencia). Incluida desde analytics, home, evidence y flow. {% endcomment %}

Desde la raíz del repo (junto a docker-compose.yml), con docker compose up en marcha. Cada carga recorre las cuatro capas en orden (la rúbrica pide ver este crecimiento Día 1 -> Día 2):

1 Origen
data_sources/dia_N/
→
2 Raw
data_lake/raw/
→
3 Processed
processed/maestro
→
4 DW
FactVentas (Postgres)

run_incremental_day ejecuta los pasos 1-4 en un solo comando. audit_pipeline muestra conteos en raw, processed y DW para la evidencia del informe.

Opción A — Un comando por día (recomendado en la demo)

1
DÍA 1
¡Copiado!
docker compose exec backend python manage.py run_incremental_day --day dia_1 --ingest-date 2026-04-01

Fuentes en data_sources/dia_1/ (~5 registros por archivo) -> ELT -> ETL maestro -> DW incremental -> auditoría.

2
DÍA 2
¡Copiado!
docker compose exec backend python manage.py run_incremental_day --day dia_2 --ingest-date 2026-04-02

Fuentes en data_sources/dia_2/ -> mismos pasos. FactVentas crece; el Día 1 se conserva.

3
EVIDENCIA
¡Copiado!
docker compose exec backend python manage.py audit_pipeline

Conteos por capa: archivos en raw/, filas del maestro por fecha, hechos en Postgres por día/mes/año. Ejecutar antes y después de cada día para el informe.

Opción B — Paso a paso (mismo flujo, más detalle)

Día 1 — datos del 2026-04-01

Copiado
docker compose exec backend python manage.py run_elt_ingest --sources-root /data_sources/dia_1 --ingest-date 2026-04-01 # ELT: origen -> raw
Copiado
docker compose exec backend python manage.py run_etl --append-master # ETL: raw -> processed (maestro)
Copiado
docker compose exec backend python manage.py load_dw --incremental --ventas-file maestro # processed -> DW
Copiado
docker compose exec backend python manage.py audit_pipeline # evidencia (captura informe)

Día 2 — datos del 2026-04-02

Copiado
docker compose exec backend python manage.py run_elt_ingest --sources-root /data_sources/dia_2 --ingest-date 2026-04-02 # ELT: origen -> raw
Copiado
docker compose exec backend python manage.py run_etl --append-master # ETL: raw -> processed (maestro)
Copiado
docker compose exec backend python manage.py load_dw --incremental --ventas-file maestro # processed -> DW
Copiado
docker compose exec backend python manage.py audit_pipeline # evidencia (captura informe)

Opcional — gráficos de respaldo

Copiado
docker compose exec backend python manage.py analyze_dw # PNG en processed/evidence/
