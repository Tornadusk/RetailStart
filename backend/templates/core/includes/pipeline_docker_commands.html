Comandos Docker (orden recomendado)

Desde la raíz del repo (junto a docker-compose.yml), con docker compose up en marcha. Cada paso alimenta el siguiente; /analytics/ y /evidence/ leen el DW y los PNG ya generados.

  1. docker compose exec backend python manage.py run_elt_ingest

    ELT: copia data_sources/ → data_lake/raw/ (misma información, nombres con fecha).

  2. docker compose exec backend python manage.py run_etl

    ETL: lee raw/, limpia y unifica → escribe CSV en data_lake/processed/ (core/etl/ingest_transform.py).

  3. docker compose exec backend python manage.py load_dw

    Carga el último ventas_unificadas_*.csv + dimensiones → Postgres (core/management/commands/load_dw.py).

  4. docker compose exec backend python manage.py analyze_dw

    Lee el DW y genera PNG en processed/evidence/ (Parte 5: gráficos de respaldo a las tablas ORM).

En Windows, las mismas rutas bajo el proyecto: …\data_sources\, …\data_lake\raw\, …\data_lake\processed\, …\data_lake\processed\evidence\.
